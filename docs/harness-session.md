# `grain_agent_harness::session`

Session persistence. Represents a conversation as a **tree** of entries linked via `parent_id`, with a `leaf_id` cursor pointing at the tip of the active branch. Supports branching, forking, moving the cursor, and compaction.

Corresponds to `packages/agent/src/harness/session/*` (excluding `jsonl-repo.ts` — on-disk JSONL persistence is not yet ported; see `grain_agent_harness::lib` doc).

中文版：[zh/harness-session.md](./zh/harness-session.md).

## Data model

```rust
pub struct SessionMetadata {
    pub id: String,            // default UUIDv7
    pub created_at: String,    // RFC3339-ish, e.g. "2024-05-21T10:00:00.000Z"
    pub extra: serde_json::Value,  // backend-private fields (e.g. file paths)
}

pub struct SessionTreeEntry {
    pub id: String,
    pub parent_id: Option<String>,
    pub timestamp: String,
    pub kind: SessionTreeEntryKind,
}
```

`SessionTreeEntryKind` (serialized as `{ "type": "snake_case", ... }`):

| Variant | Meaning |
|---------|---------|
| `Message { message }` | One `AgentMessage` (user / assistant / tool_result / custom) |
| `ThinkingLevelChange { thinking_level }` | Switch thinking level |
| `ModelChange { provider, model_id }` | Switch model |
| `Compaction { summary, first_kept_entry_id, tokens_before, details?, from_hook? }` | Compact: drop entries before `first_kept_entry_id`, keep summary |
| `Custom { custom_type, data? }` | Opaque app-level marker (not sent to LLM) |
| `CustomMessage { custom_type, content, display, details? }` | Custom-message entry that becomes `AgentMessage::Custom` when read |
| `Label { target_id, label? }` | Label another entry (`label = None` removes) |
| `SessionInfo { name }` | Session title |
| `BranchSummary { from_id, summary, details?, from_hook? }` | Summary of the branch we came from (auto-generated by `move_to(Some(summary))`) |

Each entry has `kind.type_tag()` returning the string tag (used by `find_entries(type_tag)`).

## `SessionStorage` trait

```rust
#[async_trait]
pub trait SessionStorage: Send + Sync {
    async fn get_metadata(&self) -> SessionMetadata;
    async fn get_leaf_id(&self) -> Option<String>;
    async fn set_leaf_id(&self, leaf_id: Option<String>) -> Result<(), SessionError>;
    async fn get_entry(&self, id: &str) -> Option<SessionTreeEntry>;
    async fn get_entries(&self) -> Vec<SessionTreeEntry>;
    async fn get_path_to_root(&self, leaf_id: Option<&str>) -> Vec<SessionTreeEntry>;
    async fn append_entry(&self, entry: SessionTreeEntry) -> Result<(), SessionError>;
    async fn find_entries(&self, type_tag: &str) -> Vec<SessionTreeEntry>;
    async fn get_label(&self, id: &str) -> Option<String>;
    async fn create_entry_id(&self) -> String;
}
```

Implementation requirements:

- `get_path_to_root(Some(leaf))` walks `parent_id` upward, returns entries in chronological order (root → leaf). `None` → empty.
- `append_entry` must reject duplicate ids and set `leaf` to the new entry; for `Label` entries it must sync the `labels` map.
- `create_entry_id` defaults to UUIDv7 (`grain_agent_harness::uuidv7()`).

Today: `InMemorySessionStorage` (`tokio::sync::Mutex`-protected) + `InMemorySessionRepo`. JSONL persistence will plug into the same trait — **don't change the trait shape lightly**.

## `SessionRepo` trait

```rust
#[async_trait]
pub trait SessionRepo: Send + Sync {
    async fn create(&self, id: Option<String>) -> Result<Session, SessionError>;
    async fn open(&self, metadata: &SessionMetadata) -> Result<Session, SessionError>;
    async fn list(&self) -> Result<Vec<SessionMetadata>, SessionError>;
    async fn delete(&self, metadata: &SessionMetadata) -> Result<(), SessionError>;
    async fn fork(
        &self,
        source: &SessionMetadata,
        entry_id: Option<&str>,
        position: ForkPosition,
        id: Option<String>,
    ) -> Result<Session, SessionError>;
}

pub enum ForkPosition { Before, At }   // Default = Before
```

`fork`:

- `entry_id = None` → copy all entries.
- `position = At` → new branch is `path_to_root(entry_id)`, includes the target entry.
- `position = Before` → only valid if `entry_id` is a user message (otherwise `InvalidForkTarget`); new branch is the target's parent's `path_to_root`.
- Missing entry → `InvalidForkTarget`.

## `Session` API

```rust
let session = repo.create(None).await?;
let meta = session.metadata().await;

session.append_message(agent_message).await?;
session.append_thinking_level_change("medium").await?;
session.append_model_change("openai", "gpt-4o").await?;
session.append_compaction("summary", first_kept_id, 12_345, None, None).await?;
session.append_custom("trace_marker", Some(json!({...}))).await?;
session.append_custom_message("artifact", json!("text"), true, None).await?;
session.append_label(target_id, Some("important".into())).await?;
session.append_session_name("my chat").await?;

let id = session.leaf_id().await;
let entry = session.entry(&id.unwrap()).await;
let all = session.entries().await;
let branch = session.branch(None).await;            // current leaf's path-to-root
let other = session.branch(Some(other_leaf)).await;
let label  = session.label(target_id).await;
let name   = session.session_name().await;          // last SessionInfo
```

All `append_*` return the new entry id; use this for label targets, `Compaction.first_kept_entry_id`, etc.

### Moving branches with summaries

```rust
use grain_agent_harness::session::MoveToSummary;

session.move_to(Some(other_entry_id), None).await?;     // cursor only
session.move_to(Some(other_entry_id), Some(MoveToSummary {
    summary: "user chose A".into(),
    details: None,
    from_hook: None,
})).await?;                                              // cursor + BranchSummary
session.move_to(None, None).await?;                      // back to "before root"
```

`move_to(Some(id), Some(summary))` appends a `BranchSummary` entry with `parent_id` pointing at the target. Next `build_session_context` projects this into a [`branchSummary` custom message](./harness-messages.md).

`SessionError::NotFound` if `entry_id` doesn't exist.

## `build_session_context`

Linearizes a branch into a `SessionContext`:

```rust
pub struct SessionContext {
    pub messages: Vec<AgentMessage>,
    pub thinking_level: String,             // default "off"
    pub model: Option<(String, String)>,    // (provider, model_id)
}

let ctx = session.build_context().await;        // shortcut for build_session_context(&session.branch(None).await)
```

Algorithm:

1. Walk the path in order; track latest `thinking_level` and `model` (from `ModelChange` and from assistant messages' provider/model fields).
2. Find the **latest** `Compaction` entry.
3. If compaction exists: prepend a `compactionSummary` custom message, then collect entries starting from `first_kept_entry_id`; entries after the compaction are appended in order.
4. Otherwise: collect every `Message` / `CustomMessage` / non-empty `BranchSummary` in order. `ThinkingLevelChange` / `ModelChange` / `Custom` / `Label` / `SessionInfo` are not part of `messages`.

The returned `SessionContext.messages` plugs directly into `AgentOptions { messages, .. }` or `AgentContext.messages`.

## Errors

```rust
pub enum SessionError {
    NotFound(String),
    InvalidForkTarget(String),
    Storage(String),
    Other(String),
}
```

`append_entry` rejecting duplicates returns `Storage("duplicate entry id: ...")`. `append_label` with unknown target returns `NotFound`.

## IDs / time

```rust
pub fn uuidv7() -> String;                  // UUIDv7, time-sortable
```

Timestamps use a tiny custom RFC3339 formatter (no `chrono` dependency) — fixed format `"YYYY-MM-DDTHH:MM:SS.mmmZ"`. `build_session_context` parses back to milliseconds via `parse_iso_to_ms`. No timezone handling elsewhere.

## End-to-end example

```rust
use grain_agent_harness::{InMemorySessionRepo, SessionRepo};
use grain_agent_core::{AgentMessage, TextContent, UserContent, UserMessage};

let repo = InMemorySessionRepo::new();
let session = repo.create(None).await?;
let meta = session.metadata().await;

session.append_message(AgentMessage::user(UserMessage {
    content: vec![UserContent::Text(TextContent { text: "hi".into() })],
    timestamp: 0,
})).await?;

session.append_session_name("first chat").await?;

let reopened = repo.open(&meta).await?;
let branch = reopened.branch(None).await;
assert_eq!(branch.len(), 2);   // 1 user message + 1 session_info entry

let ctx = reopened.build_context().await;
assert_eq!(ctx.messages.len(), 1); // session_info isn't in messages
```
